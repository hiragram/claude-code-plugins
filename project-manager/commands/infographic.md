# インフォグラフィック生成コマンド

ユーザーから指定された話題について、わかりやすいインフォグラフィック画像を生成します。

## 入力

$ARGUMENTS

## 手順

### 1. 入力の解析

$ARGUMENTS の内容を解析し、以下のパターンを判別してください：

- **直接指示**: トピックが直接テキストで指定されている場合
  - 例: `/infographic マイクロサービスアーキテクチャの概要`
- **URL参照**: Webページへの参照がある場合
  - 例: `/infographic https://example.com/article この記事をまとめて`
  - WebFetch で内容を取得してください
  - **重要**: 「要約して」ではなく「全セクション・全項目を構造化してリストアップして」と指示すること
- **Issue/PR参照**: GitHub Issue や PR への参照がある場合
  - 例: `/infographic Issue #123 についてまとめて`
  - `gh` コマンドで Issue/PR の内容を取得してください
- **コードベース参照**: コードや設計について説明を求められている場合
  - 例: `/infographic 認証フローについて説明して`
  - 必要に応じてコードを探索してください

### 2. 情報の全体把握（省略しない）

収集した情報を**省略せずに**全体像を把握してください：

- 元ソースのすべてのセクション・トピックをリストアップ
- 各トピックの重要度を評価（必須/重要/補足）
- 階層関係や依存関係を明確にする

### 3. 複雑度の判定と分割方針

情報量に応じて表現形式を選択してください：

| トピック数 | 推奨形式 |
|---|---|
| 5個以下 | 1枚のインフォグラフィック |
| 6〜10個 | 2枚に分割（概要編＋詳細編） |
| 11個以上 | 3枚以上、またはカテゴリ別に分割 |

**分割する場合は、ユーザーに確認を取ってから生成してください。**

### 4. 省略チェック（必須）

画像生成の前に、以下を明示してください：

```
【含めるトピック】
- トピック1
- トピック2
- ...

【省略するトピックと理由】
- トピックX: （理由: 詳細すぎる/重複/補足的）
- ...

【分割の有無】
- 1枚で収める / N枚に分割（概要編＋詳細編）
```

ユーザーが確認できるよう、**省略・分割の判断を必ず表示してから**次に進んでください。

### 5. 画像生成

Nano-Banana-MCP の `generate_image` ツールを使用してインフォグラフィックを生成してください。

プロンプト構築のポイント：
- 「Create an infographic about...」で始める
- 視覚的に明確で読みやすいレイアウトを指示
- 色使いはシンプルでプロフェッショナルに
- テキストは日本語で生成
- 情報量が多い場合は「複数カラム」「小さめのフォント」を指示

### 6. 結果の提示

生成された画像のパスをユーザーに提示してください。

## 注意事項

- 画像は一時ディレクトリに保存されます
- **網羅性 > シンプルさ**: 重要なトピックを落とすより、情報量が多い画像の方が良い
- 省略は必ずユーザーに可視化すること
- 専門用語は必要に応じて簡潔な説明を添える
- PMなど非エンジニアが理解できる表現を心がける
